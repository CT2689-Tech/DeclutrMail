# Infra snapshots

Daily JSON snapshot of every external resource DeclutrMail depends on,
generated by `scripts/infra-snapshot.sh` via the
`.github/workflows/infra-snapshot.yml` cron at 06:00 UTC.

Each file is named `YYYY-MM-DD.json`. The cron workflow commits a new
file only when the day's snapshot DIFFERS from the most recent one —
identical-state days stay silent.

## What's in here

- Cloud Run revisions (API + worker): name, image SHA, env var names
  (NOT values), traffic split.
- GCP Secret Manager: secret names + creation times (NOT values).
- Atlas migration head: current DB version per `atlas migrate status`.
- IAM bindings on `declutrmail-api` + `declutrmail-worker` SAs.
- GitHub Actions secret names (NOT values) + last-update timestamps.

## What's deliberately excluded

- Any secret VALUE — only names. This dir lives in the public repo.
- User PII / mail data — D7 / D228 compliance.
- App-layer DB rows (senders, mail_messages, triage_decisions, etc.).

## Reading a drift

```bash
diff docs/infra-snapshots/2026-06-08.json docs/infra-snapshots/2026-06-09.json
```

Any non-empty diff means infra changed between those days. Typical
expected changes: Cloud Run revision after a deploy, secret rotation,
new GH Actions secret added. Unexpected diffs are signal — read the
commit author + audit log.

## Running locally

```bash
./scripts/infra-snapshot.sh > /tmp/snapshot.json
```

The script gracefully falls back to empty/null fields when a CLI
(gcloud / atlas / gh) is unauthed locally. CI runs with full perms.
